### 编程, 和 ai编程


类比于下象棋.

任何一个人30秒都能学会象棋的前3步. 中炮, 跳马, 出車.

但是如果要把一盘棋下完, 只是下到完结, 不需要赢. 那么要学一大堆了, 马走日, 象走田, 炮打隔一位, 卒子过河才能横行, 老将不许出宫, 马腿, 象眼. 输赢, 长将, 长捉, 闲棋, 和棋...
这个步骤至少需要1小时, 再加上不少于8小时的实践, 即便如此, 你还是会忘记, 熟练掌握估计至少1个月时间.



也就是说原本编程要求你是业五水平才能参与. 那么现在, 只要你可以走到终局就能参与. 此时的胜率不取决于你的攻杀技巧(对比语言特性, 技术细节, 调试能力)了, 而是取决于布局能力(类比于架构能力).


网上不断有都市传说, 某个人不会编程, 但是, 做了几十上百个软件. 然后赚了很多钱....这个是存在的, 他在这几十上百的制作过程中学会了软件架构. 类比于象棋, 他要么本来就能走到终局, 要么在ai老师的指导下, 反复尝试, 掌握了走到终局(ai编程)的能力, 并且掌握了隐含的布局能力(架构)


##### 学习计划的可执行性

可执行的学习计划: 
第一天就可以实践, 在实践中逐步进步. 
实际上, 我们一般需要1个小时就可以实践.
真正的学习是: 10分钟之内可以实践.
0基础入门: 1分钟可以实践. 比如学习轮滑. 
1. 穿好鞋子. 这个步骤虽然可能时间较长, 但是, 他没有难度, 人人都可以.
2. 扶着固定物站起来. 这个一分钟.
3. 双手扶膝盖, 稳定住. 为了让初学者一分钟搞定这个, 一般都是用软垫/草地来练习.
    1. 稳定一分钟, 又是一分钟.
    2. 再稳定一分钟.
    3. 此时询问学员是否需要休息, 如果需要就扶固定物坐下.
4. 尝试在双手扶膝盖的情况下跺脚. 这也是1分钟之内的事情.
5. 尽量使劲跺脚. 这又是1分钟.
6. 跺脚向前走, 每一个都要保持轮子静止, 不要有轮子的滚动. 此时还在软垫/草地.
7. 到坚硬地面(不要太光滑, 平整的水泥地比较合适), 把刚才的步骤再来一遍
    1. 站起来, 如果需要的话.
    2. 扶膝静止1分钟
    3. 跺脚.
    4. 用力跺脚.
    5. 用力跺脚走路.
8. OK, 到现在应该过去1个小时了, 你可以尝试一次滑动, 保持双手扶膝的滑动.  如果是成年人(>12岁), 不做这一步, 直接到下一步葫芦步, 因为成年人很可能会受伤.
9. 尝试前进葫芦步.
10. 双手扶膝这个要在你滑动顺畅之后才能改.

0基础的核心是1分钟原则, 1分钟的原因是, 人类的短期记忆能力. 中国人普遍阈值会更高一点, 因此, 中国人的基础0学习要好一点, 甚至于, 凭借基础0学习考上大学. 在不考虑知识的结构和合理性的情况下, 考试成绩很好, 这是中国人的特性. 本质上是汉语/汉字的训练导致的能力溢出. 

非0基础学习, 有点不一样了, 他是建立在坚实的抽象基础上的, 比如某人可能会在某个领域取得完全的自由, 看啥都简单, 这就是因为他的理解非常的深入.  例如华罗庚在矩阵领域号称打洞派, 什么矩阵都能搞成稀疏的然后简化处理. 比如李白想要写诗, 喝酒就行了. 比如刘永/杜牧想要写词, 只要楚腰纤细掌中轻就行了.  比如祖冲之, 在1千多年前直接干到了实分析, 这并不能简单的认为是天赋, 这更应该认为是艰苦的思考之后, 对于概念的结构达到了'洞见'的水平, 看啥都不难. 比如马明伟搞得电磁弹射, 张益唐搞得孪生素数, 很多时候, 都是顺手的事.
